# Project Hyper Compact Overview
## Core Map
- Package: `opencode-discipline` TypeScript OpenCode plugin with bundled agent suite.
- Entrypoint: `src/index.ts` exports `DisciplinePlugin` with tools and runtime hooks.
- State engine: `src/wave-state.ts` persists plan wave state under `tasks/.wave-state/*.json`.
- Name generation: `src/name-generator.ts` creates `{adjective}-{gerund}-{noun}` plan names.
- Bundled agents: `agents/*.md` plus `agents/README.md` for install/delegation docs.
- Build artifacts: `dist/index.js` and declarations generated by `bun run build`.
## API Surfaces
- Tool `advance_wave(sessionID)` starts/advances waves with Plan-agent gating.
- Tool `accept_plan(sessionID, action, planPath?)` supports revise or Build handoff.
- Commands are loaded from `commands/*.md` and injected through `config.command` (includes `/simplify`, `/batch`, and `/deep-review`).
- Hook `tool.execute.before` blocks early plan writes and `.env` reads; enforces post-accept read-only.
- Hook `experimental.chat.system.transform` injects wave-aware discipline prompt/reminder.
- Plan kickoff nudge loop uses public APIs: asks `todowrite`, checks `client.session.todo`, retries once if missing.
- Hook `experimental.session.compacting` injects compaction resume state.
## Modules / Components
- `src/index.ts`: plugin wiring, path guards, handoff orchestration, prompt templates.
- `src/commands.ts`: markdown frontmatter loader for plugin command definitions.
- `src/wave-state.ts`: in-memory Map + file persistence + acceptance metadata.
- `src/name-generator.ts`: curated word lists and random segment picker.
- `src/__tests__/commands.test.ts`: coverage for simplify command injection and config override behavior.
- `src/__tests__/*.test.ts`: unit coverage for generator, wave-state, tools, hooks, integration.
- `agents/plan.md`: strategic planner with 4-wave discipline and plan template.
- `agents/build.md`: execution agent with phase verification and subagent delegation.
- `agents/{oracle,librarian,reviewer,designer,deep}.md`: specialized subagent definitions.
## Infrastructure / Cross-Cutting
- Runtime deps: `@opencode-ai/plugin`, `zod`; dev: `typescript`, `vitest`, `bun-types`, `yaml`.
- TS config: ESNext, bundler resolution, strict mode, declaration emit to `dist/`.
- Packaging: `files` includes `dist/` and `agents/`; ESM export map in `package.json`.
- Packaging includes `commands/` so command markdown ships with the plugin.
- Plan tracking: execution checklist maintained in `tasks/plans/vigilant-gating-sentinel.md`.
## Security Rails
- `.env` and `.env.*` reads blocked except `.env.example`.
- Plan file writes blocked before Wave 3 for active planning sessions.
- `advance_wave` and `accept_plan` reject non-Plan agent calls.
- Accepted Plan sessions become read-only for implementation writes.
